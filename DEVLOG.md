## 2026-07-07 — v3.2: 팝업 UI 전면 리디자인 + 온보딩 가이드

### 왜 — 사용자가 겪는 문제

**"검색바가 자리만 차지한다"**
팝업에서 검색을 쓰는 빈도보다 링크를 추가하는 빈도가 높다. 검색은 `Ctrl+F`로 대체 가능한데, 가장 좋은 자리를 차지하고 있었다. 링크 추가 동선이 "하단 버튼 클릭 → 폼 스크롤"로 비효율적이었다.

**"폼이 너무 길어서 팝업이 답답하다"**
워크스페이스 추가 폼에 URL, 이름, 이모지, 색상, 폴더, 만료 등 모든 필드가 한 번에 열렸다. 대부분은 URL만 넣고 바로 저장하고 싶은데, 불필요한 옵션이 화면을 가린다.

**"이모지를 매번 골라야 하나?"**
이모지 선택은 대부분 건너뛰거나 기본값을 쓴다. 선택 과정이 워크스페이스 추가의 병목이 되고 있었다.

**"온보딩 오버레이가 UI를 완전히 가려서 아무것도 못 한다"**
기존 온보딩은 풀스크린 오버레이로 각 단계를 보여줬는데, 팝업 자체가 가려져서 사용자가 혼란스럽다는 피드백.

**"서비스 아이덴티티가 없다"**
확장 프로그램의 고유 색상이 없어서 Notion 생태계 도구라는 인상이 약했다.

### 변경사항

**1. 검색바 → "링크 추가" 영역**
- 상단 검색바 제거, `＋ 링크 추가` 클릭 영역으로 교체
- 호버 시 accent 글로우 + 색상 변화로 클릭 유도
- 키보드 단축키(1~9, Enter, Escape, D)는 document 레벨 핸들러(`onGlobalKey`)로 이전
- 입력 중일 때는 단축키 무시 (input/select 포커스 체크)

**2. 컴팩트 추가 폼 + "상세 설정" 토글**
- 폼이 하단이 아닌 상단에 인라인으로 열림
- 기본 표시: URL 입력 + 저장 버튼 (1줄)
- `▸ 상세 설정` 토글로 이름, 색상, 폴더, 만료 옵션 펼침/접기
- 편집 모드에서는 상세 설정 자동 펼침
- 취소/저장 시 `＋ 링크 추가` 영역 복원

**3. 이모지 자동 할당**
- 이모지 입력 필드 완전 제거 (팝업 + 대시보드)
- 새 워크스페이스 저장 시 70종 랜덤 이모지 자동 할당
- 편집 시 기존 이모지 유지
- 재미 이모지 포함 (🦊🌮🎪🧊🪐 등)

**4. 폴더 추가 버튼 (필터 바)**

- 클릭 시 인라인 폴더 이름 입력 필드 토글

**5. 불필요 UI 제거**

## 2026-07-03 — v3.1.2: 전체 삭제, UX 디테일

### 왜 — 사용자가 겪는 문제

**"테스트 데이터 다 밀고 싶은데 하나씩 지워야 한다"**
개발 중이나 초기 세팅할 때 워크스페이스를 여러 개 등록해보고, 한 번에 리셋하고 싶을 때가 있다. 지금은 하나씩 삭제해야 해서 번거롭다.

**"전체 삭제 확인이 브라우저 기본 alert라 어색하다"**
`confirm()` 다이얼로그는 서비스 느낌이 아니다. 설정 패널 안에서 자연스럽게 확인하고 싶다.

**"대시보드에서도 링크부터 붙여넣고 싶다"**
팝업은 URL 먼저 입력하도록 바꿨는데, 대시보드 모달은 여전히 이름 필드에 포커스가 간다.

**"링크가 적을 때 팝업이 너무 작다"**
워크스페이스가 1~2개일 때 팝업이 짧아서 허전하다. 어느 정도 높이가 확보되어야 앱다운 느낌이 난다.

### 변경사항

**1. 전체 리스트 삭제 (설정 패널)**
- 설정 > 미분류 숨기기 아래에 빨간 "전체 리스트 삭제" 버튼
- 클릭 시 인라인 확인 카드로 전환 (⚠️ 경고 + 취소/삭제 버튼)
- 삭제 확인 시 워크스페이스 + 폴더 전체 초기화 후 리로드

**2. 대시보드 모달 포커스**
- 워크스페이스 추가 시 URL 필드에 먼저 포커스

**3. 팝업 최소 높이**
- `.workspace-list`에 `min-height: 320px` 추가 (링크 7개 분량)
- 워크스페이스가 적어도 일정 높이 유지

### 변경 파일

- `settings.js` — 전체 삭제 버튼 + 인라인 확인 UI
- `styles/components.css` — 위험 버튼, 확인 카드 스타일
- `dashboard.js` — 모달 포커스 URL 우선
- `popup.html` — 리스트 min-height 추가
- `_locales/en/messages.json` / `_locales/ko/messages.json` — clearAll, clearAllConfirm, clearAllYes 키

## 2026-07-03 — v3.1.1: 필터, 폴더 접기, 숫자 재설정

### 왜 — 사용자가 겪는 문제

**"폴더가 많아지면 스크롤이 끝없다"**
워크스페이스가 20개 넘어가고 폴더 3~4개로 분류하면, 팝업에서 원하는 폴더까지 스크롤해야 한다. 지금 당장 '프로젝트' 폴더만 보고 싶은데 전체를 훑어야 하는 게 불편하다.

**"폴더를 접으면 번호가 꼬인다"**
폴더 접기를 넣었는데, 접힌 항목도 1~9 단축키 번호를 차지했다. 3번을 눌렀는데 보이지 않는 항목이 열리는 황당한 상황 발생.

**"임시 링크가 언제 만료되는지 한눈에 보고 싶다"**
임시 링크를 여러 개 등록했을 때, 곧 사라질 링크부터 빨리 확인하고 싶은데 폴더별로 흩어져 있어서 찾기 어렵다.

### 추가된 기능

**1. 폴더 필터**
- 검색바 아래 필터 pill 바: 전체 + 각 폴더
- 폴더 선택 시 해당 폴더 항목만 플랫 리스트로 표시
- 검색과 동시 적용 (AND 조건)
- 폴더가 없으면 필터바 자동 숨김
- 팝업 + 대시보드 모두 적용

**2. 폴더 열고 닫기**
- 폴더 헤더 클릭으로 접기/펼치기
- ▾ chevron 아이콘 + 항목 개수 배지
- 접힌 상태 `chrome.storage.local`에 저장 (영구 유지)
- 대시보드에서 편집/삭제/드래그 버튼과 접기 동작 분리

**3. 접힌 폴더 숫자 재설정**
- 접힌 폴더 항목은 `displayOrder`에서 제외
- 키보드 단축키(1~9)가 보이는 항목만 기준으로 재할당
- 접기/펼치기 시 전체 re-render로 번호 즉시 갱신

**4. ⏳ 만료순 필터**
- 임시 링크가 있을 때만 "⏳ 만료순" pill 표시
- 남은 시간 많은 순(D-7 → D-1) → 영구 항목 맨 아래
- 폴더 구분 없이 플랫 리스트로 정렬

### 변경 파일

- `popup.js` / `popup.html` — 필터바, 폴더 접기, 숫자 재설정, 만료순 정렬
- `dashboard.js` / `dashboard.html` — 동일 기능 대시보드 적용
- `_locales/en/messages.json` / `_locales/ko/messages.json` — filterAll, expirySort 키 추가

## 2026-07-03 — v3.1 기능 추가: 임시 링크, 미분류 숨기기, 이름 자동생성

### 왜 — 사용자가 겪는 문제

v3.0을 쓰면서 세 가지 불편함이 계속 눈에 밟혔다.

**"잠깐 쓸 링크인데 영원히 남는다"**
회의 한 번, 스프린트 한 주 동안만 필요한 Notion 페이지가 있다. 끝나면 지워야 하는데 까먹는다. 워크스페이스 목록이 점점 늘어나고, 정작 매일 쓰는 링크가 밑으로 밀린다. "알아서 사라지는 링크"가 필요했다.

**"미분류가 거슬리는데 안 사라진다"**
폴더를 정리해서 모든 링크를 분류해놓았는데, 텅 빈 "미분류" 섹션이 계속 보인다. 공간을 잡아먹고, 깔끔하게 정리했다는 만족감을 깎는다. 그런데 팝업에서는 설정을 넣을 공간도 마땅치 않다.

**"링크 추가할 때마다 이름을 직접 써야 한다"**
Notion 페이지에 이미 제목이 있는데, 확장 프로그램에 추가할 때 또 손으로 이름을 입력해야 한다. URL만 넣으면 되는 걸 왜 두 번 일하게 하나. 특히 모바일처럼 빠르게 추가하고 싶을 때 이름 입력이 병목이다.

### 추가된 기능

**1. 임시 링크 (시간 지정)**
- 워크스페이스 추가/편집 시 유효 기간 선택: 영구 / 1일 / 1주일 / 직접 지정(일 단위)
- `expireAt` 타임스탬프를 workspace 객체에 저장
- 팝업·대시보드 열 때 만료된 항목 자동 삭제 (`purgeExpired`)
- 만료 예정 워크스페이스에 D-n / 오늘 만료 배지 표시
- 팝업, 대시보드 모두 적용

**2. 미분류 숨기기**
- 설정 패널에 체크박스로 미분류 섹션 표시 여부 제어
- `chrome.storage.sync`의 `settings.hideUncategorized`에 저장
- 체크 시 미분류 섹션 전체 숨김 (팝업 + 대시보드)
- `storage.onChanged`로 팝업↔대시보드 간 실시간 동기화

**3. 이름 랜덤 생성**
- 이름 필드를 선택사항으로 변경 (placeholder: "이름 (비우면 랜덤 생성)")
- 비워두면 자동 생성:
  - `notion.so` URL: 경로에서 페이지 제목 파싱 시도
  - 그 외 (`app.notion.com` 등): 랜덤 이름 부여 ("춤추는 펭귄", "저멀리 하와이" 등)
- 형용사 24개 × 명사 24개 = 576가지 조합
- `host_permissions` 추가 불필요 — Chrome Web Store 심사 영향 없음

**Pain point:** `app.notion.com` URL은 경로가 UUID뿐이라 `tab.title` 기반 자동 이름 추출이 안정적으로 동작하지 않았다. 탭 매칭, Notion API 호출 등 여러 방법을 시도했지만, 결국 랜덤 이름이 가장 단순하고 확실한 해결책이었다. 사용자가 이름을 나중에 수정할 수 있으므로, 빈칸보다 재미있는 기본값을 주는 게 낫다.

**4. UX 개선 — 폼 순서, 스크롤, 색상**
- 추가 폼 필드 순서 변경: 링크(URL) → 이모지+이름 → 색상 (기존: 이름 → 색상 → 링크)
- 포커스: 추가 버튼 클릭 시 항상 URL 필드에 포커스
- 팝업 body 스크롤바를 커스텀 스타일로 교체 (기본 브라우저 스크롤바 제거)
- 색상 팔레트에서 brown 제거 (8색 → 7색)
- 숫자 입력(직접 지정 일수)의 위아래 스피너 버튼 제거

**Pain point:** 링크를 먼저 붙여넣는 게 자연스러운 흐름인데, 이름 필드가 먼저 있어서 매번 탭키를 눌러야 했다. 추가 폼이 열리면 팝업이 길어지면서 브라우저 기본 스크롤바가 등장해 UI가 깨져 보였다.

### 법적 리서치 (유료 전환 대비)

## 2026-06-24 — 브랜딩 리뉴얼: "시간 단축"을 넘어서

### 왜

처음에는 "워크스페이스 전환을 2초로 줄여준다"가 핵심 메시지였다. 틀린 말은 아니지만, 기능 설명에 그쳤다. 타이머 앱도 시간을 아껴주고, 북마크도 시간을 아껴준다. 그렇다면 Notion Switcher는 뭐가 다른가?

고민 끝에 깨달은 것: **사람들은 시간을 아끼려고 워크스페이스를 전환하는 게 아니다.** 다음 프로젝트로 가려고, 다른 역할로 전환하려고, 새로운 아이디어를 시작하려고 전환한다. 워크스페이스 하나하나가 그 사람의 다른 가능성이다.

그래서 메시지를 바꿨다:

> **한 번의 전환으로, 당신의 다음이 시작됩니다.**

"2초 단축"이 아니라 "가능성을 여는 것"으로.

### 바뀐 것

**README 전면 개편**
- 영문: *"Every switch opens your next possibility."*
- 한글: *"한 번의 전환으로, 당신의 다음이 시작됩니다."*
- 기능 나열 중심 → 철학 + 기능 구조로 재편
- 아이콘 이미지를 README 상단에 배치

**대시보드 비주얼 업데이트**
- Falling cubes 배경 애니메이션 추가 — Notion 큐브가 부드럽게 떨어지는 효과
- 다크모드에서는 opacity를 낮추고 invert 처리
- 로고를 텍스트 "N"에서 실제 아이콘 이미지로 교체
- favicon 추가 (popup.html, dashboard.html)

**아이콘 리소스**
- icon16/48/128.png 최적화 (파일 크기 감소)
- icon16.svg, notion-cube.svg/png 추가
