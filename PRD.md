# Notion Switcher — Product Requirements Document

## 1. 제품 개요

| 항목 | 내용 |
|------|------|
| 제품명 | Notion Switcher |
| 버전 | 3.0.0 |
| 플랫폼 | Chrome Extension (Manifest V3) |
| 대상 사용자 | Notion을 다수의 워크스페이스로 사용하는 개인/팀 사용자 |
| 한 줄 설명 | Notion 워크스페이스 간 빠른 전환을 위한 Chrome 확장 프로그램 |

### 1.1 왜 만들었는가

Notion을 쓰다 보면 워크스페이스가 자연스럽게 늘어난다. 회사, 사이드 프로젝트, 개인 메모, 스터디, 프리랜서 클라이언트별... 문제는 Notion의 워크스페이스 전환 UX가 이 상황을 감당하지 못한다는 것이다.

**현재 Notion의 전환 경로:**
1. 좌측 사이드바 열기
2. 상단 워크스페이스 이름 클릭
3. 드롭다운에서 원하는 워크스페이스 찾기
4. 클릭

워크스페이스가 5개만 넘어도 매번 3~5초씩 허비한다. 하루에 수십 번 반복하면 무시할 수 없는 마찰이다. 검색도 안 되고, 순서 정렬도 안 되고, 단축키도 없다.

**핵심 가설:** "워크스페이스 전환을 2초 이내로 줄이면, 멀티 워크스페이스 사용자의 생산성 병목을 해소할 수 있다."

### 1.2 솔루션

브라우저 확장 팝업에서 **1클릭** 또는 **단축키(숫자 1~9)**로 원하는 워크스페이스에 즉시 이동. 대시보드 뷰에서 전체 워크스페이스를 한눈에 관리.

### 1.3 기능 우선순위 (왜 이 순서로 만들었나)

| 순위 | 기능 | 이유 |
|------|------|------|
| P0 | 워크스페이스 등록 + 1클릭 이동 | 핵심 가치. 이것 없으면 제품이 아님 |
| P0 | 숫자키(1~9) 바로 이동 | 키보드 유저의 킬러 기능. 마우스 없이 2초 전환 |
| P0 | 검색 | 워크스페이스 10개 넘으면 눈으로 찾기 어려움 |
| P1 | 폴더 분류 | 워크스페이스 정리 니즈. 없으면 목록이 난잡해짐 |
| P1 | 드래그 정렬 | 자주 쓰는 워크스페이스를 상단에 배치 |
| P1 | 대시보드 뷰 | 팝업은 좁음 → 전체를 한눈에 보고 정리하는 공간 필요 |
| P1 | 테마 (다크모드) | Notion 자체가 다크모드 지원 → 일관성 |
| P2 | 커스텀 색상/이모지 | 시각적 구분. 없어도 쓸 수 있지만 있으면 편함 |
| P2 | 단축키 커스텀 | 파워유저 대응. 기본값으로도 충분하지만 변경 원하는 유저 존재 |
| P2 | 다국어 (i18n) | 글로벌 배포를 위한 기반. 한국어/영어 |
| P3 | 후원 (QR) | 수익화 Phase 1. 사업자 없이 가능한 최소 모델 |
| P3 | 피드백 폼 | 사용자 목소리 수집. 제품 개선 루프의 시작점 |

## 2. 사용자 페르소나

| 페르소나 | 설명 | 핵심 니즈 |
|----------|------|-----------|
| **멀티 워커** | 회사/사이드/개인 등 3~10개 워크스페이스 운영 | 빠른 전환, 폴더 분류 |
| **프리랜서** | 클라이언트별 워크스페이스 5~20개 | 검색, 숫자키 바로 이동 |
| **팀 리더** | 팀 공용 + 개인 워크스페이스 혼용 | 정리/그룹핑, 시각적 구분 |

## 3. 핵심 기능 (v3.0.0 — 현재)

### 3.1 팝업 (Popup)

| 기능 | 설명 | 단축키 |
|------|------|--------|
| 워크스페이스 목록 | 등록된 워크스페이스를 리스트 형태로 표시 | — |
| 빠른 이동 | 숫자 1~9 입력 시 해당 순번 워크스페이스로 즉시 이동 | `1`~`9` |
| 검색 | 이름/URL/이모지로 실시간 필터링 | 자동 포커스 |
| 추가/편집/삭제 | 인라인 폼으로 워크스페이스 CRUD | — |
| 폴더 분류 | 워크스페이스를 폴더별로 그룹핑 | — |
| 새 탭 열기 | Shift+클릭 또는 ↗ 버튼으로 새 탭에서 열기 | `Shift+클릭` |
| 드래그 정렬 | 핸들(⠿) 드래그로 순서 변경 | — |
| 대시보드 이동 | 대시보드 페이지로 이동 | `D`, `Alt+Shift+N` |

### 3.2 대시보드 (Dashboard)

| 기능 | 설명 |
|------|------|
| 카드 그리드 뷰 | 워크스페이스를 카드 형태로 시각적 표시 |
| 폴더 섹션 | 폴더별 그룹핑 + 미분류 섹션 |
| 폴더 CRUD | 폴더 추가/편집/삭제 |
| 드래그 & 드롭 | 카드 순서 변경 + 폴더 간 이동 + 폴더 순서 변경 |
| 모달 폼 | 워크스페이스/폴더 추가·편집 시 모달 UI |
| 검색 | `/` 키로 검색 포커스 |

### 3.3 설정 (Settings)

| 기능 | 설명 |
|------|------|
| 테마 | 시스템/라이트/다크 3단 전환 |
| 단축키 커스텀 | 앱 내 단축키(검색 포커스, 팝업→대시보드) 변경 가능 |
| 충돌 감지 | 단축키 변경 시 기존 키/예약 키와의 충돌 경고 |
| 후원 | 토스/카카오페이 QR 코드 탭 전환 + 계좌 복사 |
| 피드백 | Tally 폼 인라인 임베드 |

### 3.4 공통 기능

| 기능 | 설명 |
|------|------|
| 다국어 (i18n) | 브라우저 언어 기반 한국어/영어 자동 전환 |
| 테마 동기화 | 시스템 다크모드 연동 + 팝업↔대시보드 간 실시간 동기화 |
| 데이터 동기화 | chrome.storage.sync — Chrome 계정 기반 기기 간 자동 동기화 |
| 커스텀 색상 | 워크스페이스별 8가지 타일 색상 선택 (자동/수동) |
| 한글 초성 추출 | 이모지 미설정 시 워크스페이스명의 한글 초성을 아이콘으로 표시 |

## 4. 기술 스택

| 구분 | 기술 |
|------|------|
| 플랫폼 | Chrome Extension Manifest V3 |
| 언어 | Vanilla JavaScript (ES5 호환) |
| 스토리지 | chrome.storage.sync |
| 스타일링 | CSS Custom Properties (Design Tokens) |
| 빌드 | 없음 (번들러 미사용, 소스 직접 로드) |
| i18n | Chrome i18n API (`_locales/`) |

### 4.1 파일 구조

```
notion-switcher/
├── manifest.json          # 확장 설정
├── background.js          # Service Worker (단축키 핸들링)
├── popup.html / popup.js  # 팝업 UI
├── dashboard.html / dashboard.js  # 대시보드 UI
├── ui.js                  # 공유 UI 헬퍼 (타일, 테마, 색상, 커스텀 셀렉트)
├── settings.js            # 설정 패널 (테마, 단축키, 후원, 피드백)
├── i18n.js                # 다국어 헬퍼
├── theme-init.js          # 테마 조기 적용 (FOUC 방지)
├── styles/
│   ├── tokens.css         # 디자인 토큰
│   └── components.css     # 공유 컴포넌트 스타일
├── icons/                 # 아이콘 (16, 48, 128px) + QR 이미지
├── _locales/
│   ├── ko/messages.json   # 한국어
│   └── en/messages.json   # 영어
└── dist/                  # 배포용 빌드
```

## 5. 데이터 모델

### 5.1 Workspace

```json
{
  "id": "1718700000000",
  "name": "회사 워크스페이스",
  "url": "https://www.notion.so/company",
  "emoji": "🏢",
  "folderId": "1718600000000",
  "colorId": 1
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | string | Y | timestamp 기반 고유 ID |
| name | string | Y | 워크스페이스 이름 |
| url | string | Y | Notion URL |
| emoji | string | N | 커스텀 이모지 |
| folderId | string | N | 소속 폴더 ID (null = 미분류) |
| colorId | number | N | 타일 색상 인덱스 0~7 (null = 자동) |

### 5.2 Folder

```json
{
  "id": "1718600000000",
  "name": "업무",
  "emoji": "💼"
}
```

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | string | Y | timestamp 기반 고유 ID |
| name | string | Y | 폴더 이름 |
| emoji | string | N | 폴더 이모지 |

### 5.3 저장소 제약

- `chrome.storage.sync`: 항목당 최대 8KB, 전체 최대 100KB
- 워크스페이스 약 200개까지 저장 가능 (현실적 사용 범위 내)

## 6. 사용자 흐름

### 6.1 워크스페이스 전환 (핵심 플로우)

```
사용자 → Alt+N (팝업 열기) → 숫자키 or 검색 → 워크스페이스 클릭 → Notion 페이지 이동
```

**목표 소요 시간: 2초 이내**

### 6.2 워크스페이스 등록

```
팝업/대시보드 → + 추가 → 이모지(선택) + 이름 + URL + 색상(선택) + 폴더(선택) → 저장
```

### 6.3 정리 및 관리

```
대시보드 → 드래그로 순서 변경 / 폴더 간 이동 → 자동 저장
```

## 7. 수익화 로드맵

| Phase | 모델 | 내용 | 상태 |
|-------|------|------|------|
| **Phase 1** | 무료 + 후원 | 전 기능 무료 배포 → 사용자 확보 + 리뷰 수집 | **진행 중** |
| Phase 2 | Freemium | 무료: 워크스페이스 10개 / Pro: 무제한 + 프리미엄 기능 | 예정 |
| Phase 3 | 팀 플랜 | Notion API 연동 + 팀 동기화 + 관리자 기능 | 예정 |
| Phase 4 | 플랫폼 | 멀티 서비스 허브로 확장 (Notion 외 서비스) | 장기 |

### 7.1 Phase 1 — 무료 + 후원 (현재)

- 전 기능 무료 제공으로 진입 장벽 제거
- QR 코드 기반 자발적 후원 (토스/카카오페이)
- 사업자등록 없이 운영 가능한 구조
- **목표:** 사용자 1,000명 + 리뷰 50개 달성 후 Phase 2 전환 판단

### 7.2 Phase 2 — Freemium (예정)

**무료 (Free)**
- 워크스페이스 최대 10개
- 폴더 최대 3개
- 기본 테마 + 단축키

**프로 (Pro)** — 월 $2.99 또는 연 $24.99
- 워크스페이스 무제한
- 폴더 무제한
- 내보내기/가져오기 (JSON 백업)
- 우선 지원

### 7.3 Phase 3 — 팀 플랜 (예정)

**팀 (Team)** — 멤버당 월 $4.99
- Notion API 연동: 워크스페이스 자동 감지
- 팀원 간 워크스페이스 목록 공유
- 관리자: 공용 워크스페이스 일괄 배포
- 팀 전용 폴더 구조 동기화

### 7.4 결제 인프라

| 시점 | 방식 | 비고 |
|------|------|------|
| 현재 | QR 송금 (토스/카카오페이) | 사업자등록 불필요 |
| Phase 2 | Chrome Web Store 결제 또는 Stripe | 사업자등록 필요 시점 |
| Phase 3 | Stripe + 대시보드 | 팀 단위 구독 관리 |
